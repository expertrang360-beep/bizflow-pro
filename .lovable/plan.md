## Part 1 — License & Plan System (NEW, primary)

### Concept

You (platform super-admin) define **Plans** with feature limits, generate **License Keys**, and sell them. Businesses redeem a key inside the app to activate their plan. Features lock/unlock based on the active license.

```text
[Super Admin Panel] → generates → [License Key] → sold/given to → [Business Owner]
                                                                       ↓ redeems in Settings
                                                          [Organization gets active subscription]
                                                                       ↓
                                                          [Feature gates check plan limits]
```

### Database (new tables)

- `plans` — id, name (Starter/Pro/Enterprise), price, billing_period (monthly/yearly/lifetime), features (jsonb of flags + limits), active
- `license_keys` — id, key (unique, format `BIZ-XXXX-XXXX-XXXX-XXXX`), plan_id, status (unused/active/expired/revoked), assigned_org_id (nullable), activated_at, expires_at, created_by, notes
- `subscriptions` — id, organization_id, plan_id, license_key_id, status (active/expired/cancelled), starts_at, expires_at, auto_renew

New role: `super_admin` (platform-level, separate from org `owner`).

### Feature flags (initial set in `plans.features`)

```json
{
  "max_branches": 1,
  "max_staff": 3,
  "max_products": 50,
  "modules": {
    "manufacturing": false,
    "ai_advisor": false,
    "payroll": false,
    "assets": false,
    "multi_branch": false
  }
}
```

### Pages & flows

- **`/admin/licenses`** (super_admin only) — Plans CRUD, generate license keys (single or batch), table with filters (status, plan), copy/export keys, revoke
- **`/settings/subscription`** (org owner) — Current plan card, expiry, "Redeem License Key" input, plan comparison
- **`/auth`** new-org signup → auto-grants 14-day Trial plan
- **`useSubscription()`** hook + **`<FeatureGate feature="ai_advisor">`** wrapper component — used to lock pages/buttons; locked state shows upgrade CTA

### Server-side enforcement

- Postgres function `check_org_feature(org_id, feature_key)` returns boolean
- RLS-friendly view `v_organization_subscription` for client reads
- Edge function `redeem-license-key` — atomic: validates key, binds to org, creates subscription, marks key active
